# 비활성 학생과 결제·교재·출석 (사용자 관점)

학생을 「삭제」하면 DB에서는 `active = false` **비활성** 처리됩니다. 아래는 **원장(관리자) 화면** 기준 동작입니다.

## 한눈에 보기

| 영역 | 비활성 후에도 보이는 것 | 숨기거나 집계에서 빼는 것 |
|------|-------------------------|---------------------------|
| **학생 목록** | 행은 그대로 남음. 상태 칩이 **비활성** | 없음 (목록에서 제거되지 않음) |
| **행 펼침 (상세)** | 등·하원, 커리큘럼·진도, 보유 교재·납부 편집 | 없음 — 과거 기록 조회·교재 납부 표시 가능 |
| **교재비 열(목록)** | `—` 표시 | 대시보드·종 알림의 **교재비 미납 집계·숫자** (활성만 집계) |
| **월별 납부 탭** | 해당 학생 **행 자체가 없음** (표는 **활성 학생만**) | 비활성 학생에 대한 월별 행·자동 청구 생성 대상에서 제외 |
| **전체 내역 탭** | 해당 학생 이름으로 **미납·완납·연체** 청구 행 표시 | 없음 |
| **대시보드 연체 건수** | `payments` 미납이면 **건수에 포함**될 수 있음 | 교재비 미납 **인원**은 활성만 |

## 권장 운영 흐름

1. **퇴원 처리** 시 학생을 비활성으로 두고, 필요하면 **퇴원일**을 입력합니다. (퇴원일이 있으면 월별 목록 규칙은 [결제 서비스 주석](../src/services/payments.service.ts)과 동일하게 적용됩니다.)
2. **미납 월 회비**를 마저 정리해야 하면 **결제 관리 → 전체 내역**에서 해당 학생 행을 찾아 납부 처리합니다. (월별 탭에는 비활성 학생이 나오지 않을 수 있습니다.)
3. **교재비**는 목록 열에서는 집계하지 않지만, 행을 펼쳐 **보유 교재**에서 납부 여부를 확인할 수 있습니다.

## 구현 참고

- 비활성화 API: `deleteStudent` → `students.active = false` — `src/services/students.service.ts`
- 교재비 집계: `TEXTBOOK_FEES.md`, `TERMINOLOGY.md`
- 월별 표 학생 목록: `listActiveStudentSummaries` (활성만)
- 전체 내역 탭: 학원 소속 전체 `payments` 행(미납·완납·연체) — `src/pages/admin/PaymentsManagement.tsx`
