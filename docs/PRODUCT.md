# PiaNote 제품 · 관리자 우선순위 (3단계)

## 관리자 핵심 화면 우선순위 (기본안)

**결정 방식:** 정량(화면별 로그) 또는 이해관계자(원장) 인터뷰가 있으면 그에 맞게 조정합니다. 없을 때의 **기본 순서**는 아래와 같습니다.

1. **학생 관리** — 일상 등록·학부모 연동·출석 핀이 운영의 중심.
2. **결제 관리** — 월 회비·청구의 수납·미납·연체(기한)는 현금흐름·CS와 직결. (교재비 납부와는 별개 — `docs/TERMINOLOGY.md` 참고.)
3. **커리큘럼·진도** — 레슨 품질·상담 설명.

**재검토 시점:** 관리자 세션 분석, 또는 분기별로 “가장 많이 여는 화면 3개”를 다시 확인합니다.

## 4단계 릴리스 노트 (요약)

- **학생 관리:** i18n 정리, 학원 미로드 시 안내, 목록 가로 스크롤(모바일), 삭제 확인·토스트 문구 통일.
- **결제:** 전체 내역 탭에서 활성 학생 없을 때 안내, 탭 전환 시 페이지 초기화, 등록/변경 토스트·빈 상태 문구 i18n, 월별 패널 로딩/터치 영역 정리.
- **커리큘럼·진도:** 초기 로딩 일원화, 진도 빈 상태·스낵바 i18n, 진도 추가 버튼 비활성 조건.

UX 횡단 규칙은 `docs/UX.md`와 동일 계열로 맞춥니다.
