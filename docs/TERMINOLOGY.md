# 용어·집계 기준 (교재비 / 월 회비·청구 / 연체)

관리자 UI(`src/i18n/ui.ts`), 서비스 코드, 본 문서에서 아래 구분을 유지합니다.

## 1. 교재비 (교재 지급·납부)

- **의미:** 학생에게 **교재를 배정**했을 때(`student_textbooks`) 그 건에 대한 **교재비 납부 여부**. 월 회비와 별개입니다.
- **미납 정의:** 배정된 행 중 `paid === false`인 행이 하나라도 있음.
- **집계·알림·학생 목록 열:** **활성 학생(`students.active === true`)만** 동일 기준. 비활성 학생은 집계·종 알림에 넣지 않고, 목록에서는 `—` 등으로 표시.
- **상세:** [TEXTBOOK_FEES.md](./TEXTBOOK_FEES.md)

## 2. 월 회비·청구 결제 (`payments`)

- **의미:** 학원이 등록한 **청구 행**. 월별 납부 탭에서 자동 생성되는 **월 회비** 청구와, 전체 내역에서 추가한 항목이 모두 `payments`에 쌓입니다.
- **연체(대시보드·종·설정 「연체 알림」):** `납부 예정일(due_date)`이 오늘보다 이전인데 `납부일(paid_date)`이 비어 있는 **청구 건 수(건)**. 숫자는 **학생 수가 아니라 미납 청구 행 개수**입니다.
- **결제 관리 UI:** 청구 행의 상태로 미납·완납·연체(`overdue`)를 표시합니다.

## 3. 비활성 학생

데이터는 삭제되지 않고 `students.active = false` 로만 바뀝니다. 결제·교재·출석이 화면에서 어떻게 보이는지는 [INACTIVE_STUDENT_UX.md](./INACTIVE_STUDENT_UX.md)를 참고합니다.

## 4. 혼동하기 쉬운 표기 정리

| 표기 | 사용처 | 비고 |
|------|--------|------|
| 교재비 / 교재비 미납 | `student_textbooks`, 학생 관리 교재 열 | 월 회비와 무관 |
| 월 회비 | 학생·학원 기본 금액 필드, 월별 청구 안내 | `payments` 금액의 주된 출처 |
| 연체 결제 · 청구 연체 | `payments` 미납·기한 경과 | **건(件)** 단위 |
| 회비 납부 | 결제 관리 표의 스위치 열 | 맥락상 **월 회비 청구**의 납부 여부 |

**쓰지 않기:** 「교육비」로 월 회비·청구 연체와 교재비를 한꺼번에 부르기 — 의미가 달라 혼동을 유발합니다.
